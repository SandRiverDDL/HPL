# 项目名称

HPL 复现与学习项目

## 语言与输出

- 始终使用中文回答。
- 始终使用中文编写代码注释。
- 回答保持简洁、结构化；无特殊需要时不写长篇解释。

## 开发环境

- 优先使用 `.venv/bin/python` 运行和检查代码。
- 训练入口优先使用 `.venv/bin/llamafactory-cli`。
- 若 `.venv` 不可用，使用 `uv run python`。
- 除非用户明确要求，禁止重建环境或重新安装依赖。
- 除非用户明确要求，禁止重新下载大型模型、WebShop 数据或搜索索引。
- 需要使用 WebShop 时，必须显式设置 Java 与 Python path，具体见 `docs/ENVIRONMENT_SETUP.md`。

## 文档分工

- `AGENTS.md`：稳定协作规则，只放长期约束。
- `docs/ARCHITECTURE.md`：项目结构、主链路与模块职责。
- `docs/STATE.md`：当前有效环境、已验证入口、已知问题与下一步。
- `docs/ENVIRONMENT_SETUP.md`：环境配置、依赖版本、WebShop 数据与 smoke 记录。

## 文档优先级

- 稳定规则以 `AGENTS.md` 为准。
- 结构分层以 `docs/ARCHITECTURE.md` 为准。
- 当前状态以 `docs/STATE.md` 为准。
- 环境复现以 `docs/ENVIRONMENT_SETUP.md` 为准。
- 若文档与实现冲突，优先修正文档并说明原因。

## 代码与设计原则

- 优先最小修改，禁止无关重构。
- 不修改与当前任务无关的文件。
- 遵循 KISS 与 YAGNI，不实现当前不需要的能力。
- 不直接修改第三方源码，除非已经定位到必须 patch 的兼容问题。
- 对第三方项目的本地 patch 必须写入文档，说明原因、影响和回滚方式。
- 新增脚本只做编排入口、参数解析和日志打印；核心逻辑应放入可复用模块。
- 共享逻辑只保留一份实现，禁止平行复制。

## 文件组织准则

- 目录优先按稳定工作流拆分，不按临时实验状态拆分。
- 同类文件一旦需要靠很长文件名区分，就应升一级目录，而不是继续平铺。
- 训练、评测和环境产物必须能从文档反查入口命令。
- 大型数据、索引、模型和缓存默认保留在项目内或本账号主要数据盘，不放系统盘。

## 训练与评测规则

- 原论文配置保留在 `train/sft/*.yaml`，不要直接覆盖。
- 本地验证优先使用 `train/sft_smoke/*.yaml`。
- 正式训练、评测或长时间 rollout 必须优先放在 tmux 会话中运行，避免阻塞当前终端和对话工具。
- smoke test 可以只使用终端 `console` 日志；正式训练、正式评测或长时间 RL 必须启用 W&B，并确认 project/name 配置后再启动。
- smoke 的完成标准是入口可启动、数据可加载、模型可前向/反向并保存最小 checkpoint。
- 正式训练前必须确认：
  - 模型路径是否为本机可用路径。
  - 数据集是否已注册到 LLaMA-Factory dataset registry。
  - GPU 是否空闲且 CUDA 可用。
  - WebShop 是否能完成最小 `reset/step`。
- 使用 W&B、MLflow 或其他外部 logger 前，应显式确认配置，避免默认上传无关实验；正式训练/评测/RL 的 W&B 例外，按上条规则必须开启。

## GPU 占卡进程规则

- 启动训练、评测或环境服务前，先确认目标 GPU 空闲情况。
- 未经用户明确要求，不自动停止未知 Python、Ray、vLLM、训练或评测进程。
- 若目标 GPU 上运行的是本项目 `scripts/train_model.py` 占卡程序，可以直接停止该进程以释放显卡，不需要额外确认。
- 因训练、评测或环境服务而停止 `scripts/train_model.py` 后，任务结束必须用默认参数恢复该占卡程序，除非用户明确要求不恢复。
- 若确需停止自己启动的临时任务，必须确认 PID、命令和工作目录。
- 临时任务结束后检查是否残留 GPU 占用。

## 终端与日志输出规则

- 查看训练、评测、下载、解压日志时，默认只提取错误、进度摘要和关键配置，避免输出大段 tqdm。
- 只有出现 OOM、Traceback、异常退出、结果缺失或用户明确要求时，才展开相关日志片段。
- 监控进度优先使用短摘要、`tail -n 100~300` 或结构化检查。
- 已确认过的模型路径、WebShop 数据路径、Java 路径、CUDA 版本等事实，后续只在口径变化或异常时复核。

## 上下文预算与检索约束

- 调查状态时，必须先用已知目录、配置文件、PID、checkpoint 路径精确定位；禁止第一步全量扫描大目录。
- 禁止对模型、索引、压缩包或大数据文件执行宽泛检索或直接读取，包括 `model.safetensors`、`*.pt`、`*.zip`、`*.jsonl`、Lucene index 文件。
- 使用 `rg` 时必须加目录和文件类型边界；禁止无边界扫 `data/`、`envs/`、`.venv/`。
- 查看日志默认只取短摘要或关键错误关键词。
- 如果一次工具输出明显超过预期，必须立即停止继续宽搜，改用更窄路径和统计命令。

## 调试与验证规则

- 出现错误时必须先定位原因，再提出修复方案，最后修改代码。
- 没有新鲜验证证据时，禁止声称链路可用、修复完成或结果可信。
- 改动训练、数据、环境、WebShop 或 LLaMA-Factory 配置后，必须同步更新相关文档。
- 最小验证优先级：
  - Python import smoke。
  - LLaMA-Factory CLI smoke。
  - WebShop import/reset/step smoke。
  - 单步训练 smoke。

## 完成定义

一次实现变更只有在满足以下条件时才算完成：

1. 代码、配置、脚本与文档口径一致。
2. 已运行相关最小验证命令并记录结果。
3. 未引入重复实现或不必要 fork。
4. 第三方依赖、数据和本地 patch 的风险已记录。

## Full Access 安全约束

- 即使开启 full access，未经用户明确要求，禁止执行删除性命令，包括 `rm`、`find -delete`、`git clean -fd`、`git reset --hard`。
- 禁止修改、移动或删除项目目录之外的文件。
- 禁止修改以下路径：`~/.ssh`、`~/.codex`、`/etc`、`/usr`、`/var`、`/opt`。
- 需要清理文件时，默认移动到项目内归档目录，不直接永久删除。
- 若写操作涉及项目外路径，必须先说明将要修改的绝对路径和目的。
- 未经用户明确要求，禁止修改 shell 配置、git 全局配置、SSH 配置和系统环境变量。

## TDD 规则

1. 本项目为复现与学习项目，禁止重度 TDD。
2. 禁止对可变训练指标做严格相等检验。
